# Fine-tuning (bonus) — QLoRA on a small open model

This directory fine-tunes a small open model (default **Llama-3.2-1B-Instruct**)
to answer Hong Kong labour questions in PoBot's grounded, cited style.

## Honest framing (read this first)

This is a **demonstration of the PEFT/LoRA skill and a cheap self-hostable
deployment path — not a quality upgrade** over the main pipeline:

- It's **knowledge distillation**: training data is generated by the **Groq 70B
  teacher**, so the 1B **student's ceiling is the teacher** — it can approach, not
  exceed it, and won't beat the 70B on hard/novel queries.
- What it *does* buy: **format discipline** (grounding, `[n]` citations, refusing
  out-of-domain questions) baked into a model small enough to **self-host cheaply,
  privately, and offline** — the "with our own model" path from the main README.
- Because the app's LLM layer is pluggable, the result **drops straight into the
  RAG pipeline** via `LLM_PROVIDER=hf_local`.

The local GPU (MX350, 2 GB) can't train an LLM, so training runs on a **free
Colab/Kaggle T4**. Everything is driven by `config.yaml` (single source of truth).

## Files

| File | Runs where | Purpose |
|---|---|---|
| `config.yaml` | — | all hyperparameters (base model, LoRA, quant, training) |
| `build_dataset.py` | **local** | corpus chunks → RAG-format instruction JSONL (teacher = Groq 70B) |
| `data/train.jsonl`, `val.jsonl` | — | generated dataset (committed) |
| `finetune_qlora.ipynb` | **Colab GPU** | 4-bit load → LoRA → SFTTrainer → eval → save adapter |
| `adapter/` | — | trained LoRA adapter (downloaded from Colab, committed) |

## Workflow

**1. Build the dataset (local).** Needs the FAISS index built and a Groq key in `.env`.
```bash
python finetuning/build_dataset.py            # full run (config: num_source_chunks)
python finetuning/build_dataset.py --limit 30 # quick/cheap run
```
Each example mirrors the exact production prompt (`build_prompt_messages` from
`src/rag.py`): `system + CONTEXT + QUESTION → grounded, cited answer`, plus
out-of-domain examples whose target is the fallback (teaches refusal).

**2. Train (Colab).** Open `finetune_qlora.ipynb` in Colab, set runtime to **T4 GPU**,
run top to bottom. It clones the repo for the dataset/config, trains the QLoRA
adapter, shows base-vs-tuned samples, and saves the adapter (~10-50 MB).
- Llama-3.2 is **gated**: request access on HF and add `HF_TOKEN` to Colab Secrets.

**3. Commit the adapter.** Unzip the downloaded adapter into `finetuning/adapter/`
and commit it.

**4. Use it in the pipeline (local).**
```bash
pip install -r requirements-finetune.txt
# in .env:
#   LLM_PROVIDER=hf_local
#   HF_ADAPTER_DIR=finetuning/adapter
python chatbot.py     # the fine-tuned model now answers through the RAG pipeline
```
(CPU inference is slow but functional — it's a demonstration.)

## Scaling up

Bigger/better result → raise `dataset.num_source_chunks`, `lora.r`, or
`training.epochs` in `config.yaml`, or switch `base_model` to `Qwen2.5-1.5B-Instruct`.
Watch validation loss to avoid overfitting.
